## 🎯 Deployment Steps (5 minutes)

### **Step 1: Create GitHub Repository**

1. Go to https://github.com/new
2. Create a new public repository named: `field-activity-dashboard`
3. Clone it to your computer:
```bash
git clone https://github.com/YOUR_USERNAME/field-activity-dashboard.git
cd field-activity-dashboard
```

### **Step 2: Copy Your Files to GitHub Folder**

Copy these files to your GitHub folder:
```
app.py                          (main dashboard)
Call data 2026.xlsx             (data file)
requirements.txt                (dependencies)
.streamlit/config.toml          (configuration)
```

### **Step 3: Push to GitHub**

```bash
cd field-activity-dashboard
git add .
git commit -m "Initial dashboard deployment"
git push origin main
```

### **Step 4: Deploy to Streamlit Cloud**

1. Go to https://share.streamlit.io/
2. Click "Create app"
3. Select your GitHub repository
4. Choose the branch: `main`
5. Set the main file path: `app.py`
6. Click "Deploy"

**That's it! Your dashboard will be live in 2-3 minutes!**

## 📞 Troubleshooting

**Q: Deploy button not showing?**
A: Make sure you're logged in to GitHub and Streamlit Cloud with same account

**Q: Getting error about missing files?**
A: Make sure all files (app.py, xlsx, requirements.txt) are in root folder

**Q: Dashboard not updating?**
A: Wait 30 seconds, then refresh. Streamlit Cloud deploys automatically

**Q: Want to use Vercel instead?**
A: Streamlit Cloud is much easier. But if you want alternatives:
- Heroku (also easy, needs credit card)
- AWS (complex setup)
- Self-hosted server (requires DevOps)
